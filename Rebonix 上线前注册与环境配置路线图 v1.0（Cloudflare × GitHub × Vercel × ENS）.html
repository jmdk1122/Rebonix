2. GitHub 与 Vercel 注册与连接步骤

  1. 创建 GitHub 账户 → 登录后新建仓库 Rebonix。
  2. 上传所有 HTML / 文档文件，可设为私有或公开。
  3. 在 Vercel 使用 GitHub 登录，点击「Import Project」。
  4. 选择仓库 Rebonix → Framework 选择「Other」→ 部署。
  5. 部署后生成地址：例如 https://rebonix.vercel.app。
# 测试命令（验证部署） $ curl -I https://rebonix.vercel.app

3. Cloudflare DNS → Vercel 绑定模板

CNAME www cname.vercel-dns.com Proxied=ON A @ 76.76.21.21 Proxied=ON

完成后，在浏览器中访问 https://www.rebonix.io 即可跳转到你的 Vercel 部署页面。确保 SSL 模式设为 Full (Strict)，并启用「Always Use HTTPS」。

4. ENS 域名与链上确权

ENS（Ethereum Name Service）是链上域名系统，用于确立 Rebonix 的 Web3 身份。

  1. 访问 ens.domains
  2. 连接 MetaMask 钱包，搜索并注册：
    rebonix.eth 与 fomocare.eth
  3. 在 ENS 设置中绑定你的主域名：
    contenthash → rebonix.io
  4. 未来可在白皮书页脚加入链上验证标签：
🔗 Verified on-chain: rebonix.eth

6. 安全检查与最佳实践

  • Cloudflare：开启 WAF、Bot Fight Mode、防火墙规则。
  • Vercel：启用访问日志与 Analytics，防止滥用。
  • GitHub：设置 2FA，定期备份仓库到 NAS。
  • 邮箱：添加 SPF / DKIM / DMARC 记录，防止钓鱼。
  • 建议创建 README_Rebrand.txt 与 CHANGELOG.md 归档迁移历史。
