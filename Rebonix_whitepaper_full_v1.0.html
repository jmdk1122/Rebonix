第1章：前言与愿景

Rebonix 诞生于行为金融和区块链结合的窗口期。它关注的不再是杠杆、收益，而是人类的情绪风险：后悔、错过、恐惧（FOMO）。

在传统金融中，风险可以被对冲，但情绪风险无人承担。Rebonix 的使命是：让情绪被定价，让后悔可以被结算。

它不是博彩，也不是保险，而是一个介于两者之间的“确定性幻想”层（Certainty Fantasy Layer）。

第2章：用户痛点与问题定义

  1. 散户无法应对行情剧烈波动：长期看好却被短期恐惧击退。
  2. 期权与衍生品太复杂，不具备全民化认知门槛。
  3. 心理账户未被定价：市场里只有风险价格，没有心态价格。
  4. DeFi 产品理性、用户行为感性，供需失衡。

Rebonix 让“后悔值”第一次成为金融变量。

第3章：Rebonix 概念与系统架构

系统定义

Rebonix = Emotional Finance Layer
一个参数化、去中心化的情绪风险清算系统。

三层结构

  • Ticket Layer（用户层） — 发行看涨/看跌票据；
  • Vault Layer（缓冲层） — 收取保费、支付赔付、维持风险金池；
  • Hedge Layer（对冲层） — 外部对冲仓平衡残差风险。
用户买票 → 保费入池 → Oracle 判定 → 自动赔付 → 风控回写

核心创新特征：小额、高频、参数化、可验证、情绪驱动。

第4章：产品双轨结构

轨道产品心理价值周期内核结构
ASmart Exposure Vault（聪明仓）长期看多，怕波动30–90天定投 + 保险组合
BParametric Price Ticket（参数化票据）短期行情体验7天看涨/看跌票据 + Oracle赔付

A 层是长期心态的防抖器，B 层是短期市场情绪的出口，两者共同构成 Rebonix 的闭环生态。

第5章：市场对照与创新定位

Rebonix 在“行为金融 × Web3”的交叉点上定位自己。对比 PoolTogether、Dopex、Polymarket、InsureDAO 等，它既非博彩，也非保险，而是“确定性幻想”产品。

  • 传统金融卖理性工具；
  • DeFi 卖投机机会；
  • Rebonix 卖心理确定性。

第6章：演化路径与情绪金融新层

从结构性票据（TradFi）到链上结构化 Vault（DeFi），再到参数化情绪票据（Rebonix），这是金融演进的第三阶段。

  1. 1980s–2000s：银行设计期权结构票据。
  2. 2019–2023：Dopex / Ribbon 把结构票据上链。
  3. 2025–：Rebonix 把结构逻辑情绪化、参数化。

第7章：商业模式与可持续盈利设计

核心理念

以信任为先，盈利后置。所有保费先进入赔付金池，资金池安全稳定后再收取服务费。

三阶段盈利路径

  1. Phase 1：信任建设期（无抽成）
  2. Phase 2：生态稳态期（5% 技术服务费）
  3. Phase 3：生态协作期（API + 品牌 + 代币）

代币经济模型

  • 代币符号：RXN
  • 功能：治理投票、抵押、流动性激励、回购销毁。

第8章：风险控制与对冲体系设计

三层防线：小额高频 → 用户票据内部对冲 → 外部期权仓残差对冲。

系统风控指标包括：赔付率p、内部对冲率ΔH、缓冲覆盖率BCR、净敞口NE、系统健康指数S。

if imbalance_ratio > 0.25 or expected_payout > buffer_balance * 1.5:
    hedge_size = expected_payout - buffer_balance
    buy_put_option(exchange="Deribit", amount=hedge_size, duration="7d")

最终目标：系统净风险趋近于零。
