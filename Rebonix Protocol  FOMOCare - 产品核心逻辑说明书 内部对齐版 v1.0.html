Rebonix Protocol / FOMOCare — 产品核心逻辑说明书（内部对齐版 v1.0）

0. TL;DR（给忙人）

  • 我们卖的不是“中奖失败保险”，而是“错过机会的行为保险（Behavioral Insurance）”。
  • 用户只需提交一个可验证的“计划”（Plan Commit）并支付小额保费 F，之后与其是否真的去交易无关。
  • 若在“观察窗口 Δt”内，市场价格路径验证该计划本可赚钱且超过阈值 τ，触发定额赔付 G（如 k×F）。
  • 本质：我们是期权卖方（卖一张“错过机会触发的 One-Touch/Lookback 型参数化赔付”），保费 = 收入，触发概率低 + 赔付封顶 ⇒ 正期望。
  • 两层品牌：Rebonix（协议层）：专业风控与结算；FOMOCare（应用层）：面向散户/KOL 的“情绪对冲”入口。

1. 产品定义与边界

1.1 我们在卖什么

  • 行为保险（Parametric Behavioral Insurance）：为“计划未执行/提前离场/未加杠杆”导致的机会成本提供参数化（不依赖理赔材料）的自动赔付。
  • 参数化意味着：只要链上可验证的计划参数 + 公开价格喂价满足触发条件，即自动结算，不看用户是否真的下单。

1.2 关键对象与变量

计划（Plan）四元组：P = (标的 S、方向 D∈{Long,Short}、阈值 τ、窗口 Δt)。可选扩展：冷却期 c、参考价 s₀（commit 时取 TWAP/Median）、最大赔付 G 上限、费用 F。

  • 触发条件（示例）：多头：在 [t₀+c, t₀+c+Δt] 内，价格涨幅 ≥ τ（相对 s₀）；空头：同窗内，价格跌幅 ≥ τ。
  • 赔付规则：常用定额（如 G = k×F），也可做“阶梯定额”。

1.3 核心差异点

  • 与真实交易解耦：用户是否真的下单、仓位多大，都不影响理赔判定（我们只看“计划 + 价格路径”）。
  • 标准化、可嵌入：任何交易平台/钱包/社区都可嵌入“计划卡片 → 付费 → 自动结算”的组件。

2. MVP 产品形态（首发 2–3 个就够）

所有形态均遵循“计划承诺 → 冷却期 → 观察窗口 → 触发即赔”的统一流程。

2.1「未下单险」Plan-but-No-Trade

  • 用户心智：我看多 ETH，但临门一脚没下；结果 24h 涨了 5%。
  • 触发：Long 且 24h 涨幅 ≥ 5%（或自定义 τ、Δt）。
  • 赔付：G = 10×F（例）。
  • 价值：门槛最低、教育成本小、最适合冷启动投放。

2.2「错过杠杆险」Missed-Leverage

  • 用户心智：本来想上 3×，临时怂了；如果真上杠杆，收益会更高。
  • 触发：方向正确且价格触达“等效杠杆收益”阈值（用价差阈值近似）。
  • 赔付：G = k×F（k 随 L、τ、Δt 动态）。
  • 价值：强情绪共鸣，客单价与毛利更高。

2.3「平仓后涨回险」Exit-Then-Rebound（可选）

  • 用户心智：卖飞了，结果短时间内又涨回 ≥ X%。
  • 触发：s₀ 取自“计划的退出价”（不要求真实下单），Δt 内上涨 ≥ τ。
  • 备注：若想完全不依赖真实交易，可让用户提交“计划退出价”作为 s₀。

后续可扩展：「止损/止盈悔单险」「组合收益缩水险（NFT/LP/指数）」等。

3. 用户路径与链上流程

  1. 创建计划：填 S、D、τ、Δt（可有预设模板：+3%/24h、+5%/48h…）。
  2. 计划承诺（Commit）：EIP-712 签名或上链存证；扣保费 F（USDC）。
  3. 冷却期 c：防止用户在突发波动后“追着 K 线买保险”。
  4. 观察窗口 Δt：调用 Chainlink/Pyth（TWAP/Median）按方向与阈值判断是否触发。
  5. 结算：若触发，合约自动向用户地址支付 G（k×F）；否则无赔付。
  6. NFT 保单（可选）：每份计划铸造 NFT 作为凭证，可用于二级市场或 gamification。

4. 定价与盈利（保持正期望）

4.1 定价框架（原则导向）

  • 核心：保证 F > p(trigger) × G。
  • p(trigger) 估计方法：简法（历史波动率 + Δt + 单边阈值 τ）、强化（One-Touch/障碍型近似或蒙特卡洛）。
  • G（赔付）：建议定额上限（G = k×F，k 依参数分段）。
  • F（保费）：基础费率 × 风险因子（σ、τ、Δt、方向拥挤度、资金池健康度）。

4.2 典型参数（建议起始）

参数建议区间备注
阈值 τ3% / 5% / 8%预设档，便于营销与比较
窗口 Δt12h / 24h / 48h与波动率耦合，Δt 越长 p 越高
冷却期 c10–15 min防追涨/插针套利
倍数 k5× / 10× / 20×随 τ、Δt、σ 动态；上限 20×
单保单赔付上限$100–$300控尾部、控黑天鹅
单地址日限额$300–$1,000防刷子与相关性聚集

4.3 单保单期望

E[利润] = F − p(trigger) × G。触发概率低 + 赔付封顶 ⇒ 毛利接近 F（扣渠道分成）。

5. 风控与反规避

  • 时间控制：冷却期后才进入观察窗；支持“宽松开始（Grace Start）”与“固定快照”。
  • 价格可信度：双预言机（Chainlink/Pyth），TWAP/Median 去噪；对极端蜡烛设异常过滤。
  • 相关性约束：同一标的/同方向/同参数的敞口分层计量，触发“拥挤度阈值”自动提价或限量。
  • 额度与分层：资金池分层（Senior/Junior），黑天鹅暂停开关；单地址/单标的/单窗口限额。
  • 计划承诺反舞弊：EIP-712 Commit-Reveal；计划锁定不可改；撤销仅退部分费用；相似度检测与冷却再购买。

6. 合作对接（怎么嵌）

  • CEX/交易 App：计划卡片 SDK（调用我们合约），他们拿渠道分成；UI 一键“创建计划 → 支付 → 等待结算”。
  • 钱包/聚合器：在行情页/提醒页嵌“错过对冲”按钮。
  • KOL/社区：推广码与返佣；KOL 可发“计划模板包”。
  • DeFi/衍生品协议：把“行为保险”当增值模块，提升留存与净入金。

8. 合规与披露（高层原则）

  • 性质定位：参数化保障合约（parametric payout），信息披露充分；非储蓄/非投资承诺。
  • 地域与准入：设地理屏蔽与 KYC 模式开关（由合作方/前端决定适用范围）。
  • 宣传口径：严禁“稳赚/提高胜率”，只强调情绪对冲与机会成本缓释。
  • 法律咨询：关键市场（US/EU/SG/HK）走本地律所意见（统一法律包）。

10. FAQ（统一口径）

Q1：用户不下单也能赔？ 是。我们只验证“计划 + 价格路径”，与真实交易解耦。

Q2：会被“追 K 线买保险”薅吗？ 冷却期 + TWAP/Median + 阈值/窗口 + 拥挤提价/限额，降低此类风险。

Q3：怎么定价才不亏？ 保证 F > p×G；p 用历史/隐含波动 + Δt + τ 估计，并随资金池健康度动态调价。

Q4：赔付资金从哪来？ 保费净收入 + LP 池；G 定额封顶，池子做分层与限额。

Q5：和“预测市场/博彩”差别？ 我们不对输赢赔率，而是对“错过的路径事件”付定额；属于参数化保障。
