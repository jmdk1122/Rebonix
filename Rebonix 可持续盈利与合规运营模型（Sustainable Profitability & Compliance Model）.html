第七章：商业模式与可持续盈利设计

核心理念

以信任先行，不靠抽水盈利。Rebonix 的初期策略借鉴 PoolTogether：理念正确、风险低、结构透明、先赢信任再谈盈利。

阶段性盈利路径

  1. Phase 1：信任建设期（无抽成） — 所有保费收入进入赔付金池。
  2. Phase 2：生态稳态期（5% 技术服务费） — 当资金池规模充足后，启动少量服务费。
  3. Phase 3：生态协作期（API + 联合品牌 + 代币经济） — 多元化收入模型。

与交易所协同路径

  • 联合品牌产品：如「OKX × Rebonix 聪明仓」。
  • 投顾 API：在交易所定投页面出现「添加 FOMO Cover」。
  • 获取匿名定投数据用于算法优化，提升赔付稳定性。

代币经济模型（Tokenomics）

  • 代币符号：RXN
  • 功能：治理投票、对冲抵押、流动性激励、回购销毁。
  • 来源：保费抽成 / 合作收入 / 国库激励。
  • 目标：长期稳定的价值锚，而非短期投机。

第八章：风险控制与对冲体系设计

风险哲学

“Rebonix 不靠赢用户的钱，而靠让风险归零。”
平台不押方向，核心目标是使系统的净敞口趋近于零。

三层风险防线

层级机制说明
第一层：票据层小额 / 高频 / 固定赔付单票风险极低，自然分散
第二层：内部对冲层UP / DOWN 用户票据互抵系统自动平衡方向分布
第三层：外部对冲层期权仓 (Deribit / OKX / Lyra)内部不平衡时由衍生品对冲残差

动态对冲算法

if imbalance_ratio > 0.25 or expected_payout > buffer_balance * 1.5:
    hedge_size = expected_payout - buffer_balance
    buy_put_option(exchange="Deribit", amount=hedge_size, duration="7d")
else:
    continue_issue_tickets()

关键风控指标

指标定义目标值
赔付率 p实际赔付 / 总票数< 10%
内部对冲率 ΔH|UP票金额 − DOWN票金额| / 总票金额< 25%
缓冲覆盖率 BCRBufferVault / 99%分位赔付> 1.2
净敞口 NEΣ(潜在赔付 − 对冲仓收益)< 5%
系统健康指数 SS = 1 − (p + ΔH/100 + 1/BCR)> 0.8

Oracle 与透明结算层

  • 价格取样：TWAP / Median。
  • 所有赔付、对冲记录链上可查，可审计。

风险哲学总结：Rebonix 的核心价值不在赚取保费，而在于打造一个「风险平衡的市场机制」，让情绪风险通过参数化建模被清算。
