从产品到基础设施：Emotional Finance Layer as a Service

定位 · 不做 App，做层

Rebonix = Emotional Finance Layer（EFLaaS）。我们不与交易所/应用争入口，而是以 API / SDK 形式，为其用户提供“心理对冲（情绪保险）”能力。

一句话：Don’t hedge the market. Hedge yourself.

FOMOCare™ 防抖器 FUD Shield™ 心理保险 EmoX Engine™ 拥挤度定价

商业模式（B2B2C）

  • 对 交易所 / 钱包 / DeFi 协议：提供嵌入式“心理对冲”组件；按成交额抽成或按调用量计费。
  • 对 终端用户：在原有开仓/抵押界面出现 “添加心理对冲（$5 起）” 勾选项。
  • 结算：参数化赔付，链上透明，批次化清算。

护城河

  • 算法护城河：行为金融 × AI 拥挤度调价（EmoX），依赖心理数据与动态参数，难以复制。
  • 信任护城河：去中心化准备金与链上清算，CEX 难以等价透明。
  • 合规护城河：参数化保险叙事，新加坡主体；非博彩、非二元期权。
  • 整合护城河：深度嵌入合作方流程，替换成本高。

最小可用接口（MVP API）

POST /api/v1/fomocare/hedge
Body:
{
  "user": "hashed_id",
  "asset": "BTC",
  "side": "long|short",
  "notional": 10000,
  "tenor_days": 7,
  "risk_profile": "conservative|balanced|aggressive"
}

Response:
{
  "premium_usd": 12.4,
  "cover_profile": "drawdown-15%-cap",
  "payout_max_usd": 800,
  "policy_id": "rbx_20251020_xxxxx"
}

SDK：JS（前端弹窗组件）、Python/Go（后端撮合）、Solidity（链上结算适配器）。

技术路线

  • EmoX Engine：基于波动率/拥挤度/资金费率/链上行为的动态保费。
  • 批次化发行：统一时间窗聚合，降低对冲与清算成本。
  • 清算与储备：链上可验证的准备金与赔付路径。

合规模块

  • 参数化保险条款（非保证收益）。
  • 披露：风险、赔付规则、准备金规模。
  • 新加坡主体 + KYC/AML 适配（合作方负责终端 KYC）。
