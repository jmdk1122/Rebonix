Rebonix 风控架构：Safe Launch 版本（白皮书专用章节）

0. 章节目的

本章节定义 Rebonix 在早期试运行阶段（Safe Launch）的三层防线：限量发行、动态赔率、残差对冲。目标是确保在 AI/ML 行为模型尚未完全成熟前，系统的赔付风险被上限封顶且可验证。

1. 三层防线（概览）

层级机制功能关键指标
第一层发行量上限（Ticket Cap）限制总赔付上限，控制系统最大敞口Cap、MaxPayout、BufferRatio(BCR)
第二层动态赔率（Adaptive Pricing）基于拥挤度调整票价/赔付，提升内部对冲率p、E[π]/F、ΔH（内部对冲率）
第三层残差对冲（Residual Hedge）对极端单边行情进行外部对冲（期权/现货）Hedge Ratio h、滑点/费率成本

2. 限量发行（Ticket Cap）

设首次发行 N = 10,000 张票据，每张面值 F，最大赔付 P_max 固定。则早期运行期内系统最大赔付为：

MaxPayout = N × P_max

配合风险缓冲金 Buffer 与资金池 Pool，定义：

BCR = (Pool + Buffer) / MaxPayout

Safe Launch 要求 BCR ≥ 1.2（可根据实际调整至 1.3–1.5），确保即便出现极端单边赔付，系统亦可履约。

3. 动态赔率（Adaptive Pricing）

3.1 拥挤度与价格映射

令 q 为当前 UP 方向的占比（拥挤度），基础票价 F，价格调整函数 f(q)：

TicketPrice_up   = F × f(q)
TicketPrice_down = F × f(1 - q)

在初版中，为简化实现与增强可解释性，采用区间分段而非连续函数：

UP:DOWN 拥挤度票价倍率 f(q)赔付调整说明
50:50 ±10%1.00×100%基准区间
65:351.20×90%轻度拥挤
80:201.50×75%明显拥挤
≥90:102.00×60%极端拥挤（同时触发外部对冲阈值）

3.2 目标函数

通过价格和赔付的双变量调节，提升内部对冲率 ΔH，并将 p（赔付概率目标）控制在 < 8% 的区间。

实现要点：前端需透明展示价格/赔付随拥挤度变化的曲线或阶梯，形成“情绪—价格”的负反馈，抑制单边参与。

4. 残差对冲（Residual Hedge）

当拥挤度达到极端阈值（如 ≥90:10）或 ΔH 低于安全线（如 < 0.6）时，触发外部对冲比例 h：

h = α × (NetExposure / (Pool + Buffer))    # α ∈ [0.5, 0.7]

执行路线：

  1. 优先使用期权（Deribit/OKX）进行短期对冲；
  2. 单笔对冲设最小规模阈值与时间窗，降低费率与滑点；
  3. 所有对冲事件链上回写（时间戳、方向、数量、成本）。

5. 指标与告警

指标定义告警阈值动作
p周期赔付概率> 8% 连续 3 期提高价格区间 / 降低赔付区间
E[π]/F单位面值期望收益< 0.4 连续 2 期提高基准票价 F 或缩小赔付
ΔH内部对冲率< 0.6触发外部对冲
BCR缓冲比< 1.2暂停发行 / 提升 Buffer

6. 用户体验与透明度

  • 展示「当前拥挤度、价格倍率、潜在赔付」的简明卡片；
  • 公示每期的概率与赔付统计图（周报）；
  • 公开对冲记录与费用占比，构建「非庞氏」信任锚。

7. 版本演进

  1. v1（本章）：分段定价 + 固定阈值 + 手动参数校准；
  2. v2：引入 ML 对拥挤度进行短期预测，自动调节阶梯；
  3. v3：强化学习（RL）联合优化价格与赔付，目标最小化净敞口与波动；
  4. v4：全自动残差对冲编排 + 费用最优执行。

8. 小结

通过「限量发行 + 动态赔率 + 残差对冲」三层防线，Rebonix 在缺少成熟 AI 行为模型的情况下依旧可以实现可控可审计的安全启动。此架构将用户情绪转化为价格信号，形成负反馈闭环，并将极端风险转移至受限范围内。
